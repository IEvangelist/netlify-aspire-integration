# Quickstart.AppHost (C#)

A minimal C# AppHost that boots a single Astro site and shows off the
`PublishAsNetlifySite("dist", authToken)` convenience overload added in
[`Aspire.Hosting.Netlify`](../../src/Aspire.Hosting.Netlify).

## Run locally

```sh
aspire run --apphost samples/Quickstart.AppHost/Quickstart.AppHost.csproj
```

Aspire starts the Astro dev server alongside the dashboard. Open the dashboard URL
that's printed to your terminal to inspect the resource tree.

## Deploy to Netlify

```sh
aspire deploy --apphost samples/Quickstart.AppHost/Quickstart.AppHost.csproj
```

Authentication precedence (highest first):

1. The `netlify-token` parameter (resolved via user secrets / configuration).
2. The `NETLIFY_AUTH_TOKEN` environment variable.
3. Interactive `ntl login`.

> **Tip.** The directory-only overload sets `NoBuild = true` for you — it deploys
> a directory you've already produced (here, the `dist/` folder generated by
> `npm run build`). For unbuilt content, use the full `NetlifyDeployOptions`
> overload instead.
