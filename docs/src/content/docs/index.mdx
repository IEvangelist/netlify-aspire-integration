## Three lines of C#

<Tabs>
  <TabItem label="C#" icon="seti:c-sharp">
```csharp
var builder = DistributedApplication.CreateBuilder(args);

builder.Pipeline.AddNetlifyDeployPipeline();

builder.AddJavaScriptApp("astro", "../astro")
    .PublishAsNetlifySite("dist");

builder.Build().Run();
```
  </TabItem>
  <TabItem label="TypeScript" icon="seti:typescript">
```ts
import { Aspire } from "./.modules/aspire";

const builder = Aspire.createBuilder();

builder.pipeline.addNetlifyDeployPipeline();

builder.addJavaScriptApp("astro", "../astro")
  .publishAsNetlifySite({ dir: "dist", noBuild: true });

builder.build().run();
```
  </TabItem>
</Tabs>

Then:

```sh
aspire deploy
```

That's it. The pipeline finds the Netlify CLI (auto-installs if needed),
resolves your auth token, and ships your build to Netlify.

## Why this integration?

<CardGrid>
  <Card title="Pipeline-aware" icon="rocket">
    Hooks into Aspire's deploy pipeline. `aspire deploy` is the one command —
    no bespoke shell scripts, no hand-rolled GitHub Actions.
  </Card>
  <Card title="Multi-language" icon="translate">
    Annotated with the
    [Aspire Type System](https://aspire.dev/architecture/multi-language-architecture/)
    so a TypeScript AppHost gets fully-typed `publishAsNetlifySite(...)`,
    `withNpmCommand`, and `withNpmRunCommand` exports.
  </Card>
  <Card title="Secret-safe auth" icon="approve-check-circle">
    Auth token resolves from a parameter, then `NETLIFY_AUTH_TOKEN`, then
    interactive `ntl login`. Site IDs and tokens are redacted from logs.
  </Card>
  <Card title="Six framework demos" icon="puzzle">
    [Angular](/netlify-aspire-integration/frameworks/angular/),
    [Astro](/netlify-aspire-integration/frameworks/astro/),
    [Next.js](/netlify-aspire-integration/frameworks/next/),
    [React](/netlify-aspire-integration/frameworks/react/),
    [Svelte](/netlify-aspire-integration/frameworks/svelte/), and
    [Vue](/netlify-aspire-integration/frameworks/vue/) — all wired up in the
    repo under `samples/`.
  </Card>
  <Card title="OIDC-published" icon="seti:lock">
    No long-lived NuGet API key. Releases are published via
    [Trusted Publishing](/netlify-aspire-integration/guides/ci-cd/) on every
    tagged release.
  </Card>
  <Card title="MinVer-driven semver" icon="setting">
    Versions come straight from git tags via
    [MinVer](/netlify-aspire-integration/guides/versioning/) — no
    `Version.props`, no manual bumps.
  </Card>
</CardGrid>
